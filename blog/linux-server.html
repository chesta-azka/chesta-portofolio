Optimasi Linux untuk High-Traffic_

Chesta Azka // March 02, 2026

Banyak pengembang mengabaikan konfigurasi sistem operasi saat membangun aplikasi. Padahal, secepat apapun kode lo, kalau **kernel Linux** lo kaku, sistem bakal tumbang di bawah ribuan request serentak.

1. Tuning File Descriptors (Limits)

Secara default, Linux membatasi jumlah file yang bisa dibuka (termasuk koneksi jaringan). Untuk server high-traffic, kita harus menaikkan limit ini agar tidak terjadi error `Too many open files`.

# Periksa limit saat ini
ulimit -n

# Naikkan limit di /etc/security/limits.conf
* soft nofile 65535
* hard nofile 65535

# Apply perubahan
sysctl -p

2. Optimasi TCP Stack

Masalah paling umum pada server yang sibuk adalah antrean koneksi yang penuh. Kita perlu mengoptimalkan parameter TCP agar sistem lebih cepat melepaskan koneksi yang sudah tidak terpakai (`TIME_WAIT`).

# Edit /etc/sysctl.conf
net.ipv4.tcp_fin_timeout = 15
net.ipv4.tcp_tw_reuse = 1
net.core.somaxconn = 4096
net.ipv4.tcp_max_syn_backlog = 8192

3. Penggunaan Nginx sebagai Reverse Proxy

Menggunakan Nginx bukan cuma soal routing, tapi soal **Buffering** dan **Caching**. Pastikan lo mengaktifkan Gzip compression untuk mengurangi payload data ke klien.

Kesimpulan

Infrastruktur yang kuat adalah fondasi dari user experience yang mulus. Tanpa optimasi di level OS, hardware mahal sekalipun akan terasa lambat.
